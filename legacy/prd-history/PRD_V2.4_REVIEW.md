## 5. 分角色评审清单

### 5.1 产品

- [ ] 核心用户、JTBD、P0/P1 和成功指标可执行。
- [ ] 多次轻触点与基础报告/片尾深挖关系无歧义。
- [ ] 所有影响实施的待决策已关闭或明确延期。
- [ ] 未把 PM、GitHub Pages、fixture 或候选内容写成上线事实。

### 5.2 设计与无障碍

- [ ] 48vh、无蒙层、至少 52% 视频可见符合目标体验。
- [ ] 六类模板在单动作/12 秒/44px/键盘/200% 字体约束下可实现。
- [ ] 财包与作者身份始终分离；字幕、轨迹和系统安全区有避让。
- [ ] 片尾复杂内容不会重新挤入播放中半屏。

### 5.3 研发与架构

- [ ] Vue/Vite/Pinia/Express 单栈、内容流/运行流和恢复策略可实施。
- [ ] 已实现与 P0 目标字段区分准确。
- [ ] 完整版本向量、ReviewManifest 与批准/发布分离可落到 Schema/API/CLI。
- [ ] Provider、规则方向、Prompt 注入和故障边界闭合。
- [ ] 公网环境鉴权/限流/日志/密钥与回滚有 Owner。

### 5.4 测试

- [ ] F-01..F-20 均能映射页面/模块、事件和测试。
- [ ] PM adapter、Review Candidate/draft 越权、发布回滚有负向用例。
- [ ] 六类时间轴、不停播、4/45、恢复、报告与安全有自动化门禁。
- [ ] 未执行的真实 Provider、授权内容、人工审核与 PDF 视觉验收均明确未通过。

### 5.5 内容与财经审核

- [ ] 首包学习目标与视频真实覆盖一致，不注入股票/黄金能力。
- [ ] 概念、因果边、条件、反例、选项、方向与时间码审核可操作。
- [ ] CoverageReport 是缺口清单，不是内容分数。
- [ ] 规则未知组合返回 `insufficient`，模型不改变方向。

### 5.6 法务、版权与来源

- [ ] 视频/作者授权、用途、范围、期限、下架和媒体指纹字段充分。
- [ ] 公开页面探测、OAuth 元数据与可分析媒体的界线准确。
- [ ] 未授权 PM 媒体不会复制进产品仓或公开发布链。
- [ ] 底座 GPL/README 限制与形象/截图等第三方资产已有处理结论。

### 5.7 安全与隐私

- [ ] `.env`、Provider 费用、真实调用开关和密钥轮换边界可接受。
- [ ] ASR/OCR/视频不可信输入与提示注入防护覆盖每个 LLM 阶段。
- [ ] 原始语音、媒体派生物、复述和日志保留满足最小化。
- [ ] 匿名 API 的鉴权、限流、配额、CORS 和网络绑定有明确门禁。
