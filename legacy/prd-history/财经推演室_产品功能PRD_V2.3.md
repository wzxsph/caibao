# 1. 项目现状与来源裁决

[现状事实] PM 仓库为 React 19 + Next/Vinext + Cloudflare 的独立原型。它包含约 4 分 16 秒本地视频、版本化 `VideoContentPackage`、章节、字幕线索、5 个概念、3 条因果边、6 个节点、浏览器语音入口和报告配置。

[现状事实] 内容包 ID 为 `fed-rate-global-capital-001`，版本为 `2026-07-22.2`，状态为 `draft`。5 个概念与 3 条因果边均未完成审核；源码明确说明 OCR、文字与时间戳仍需人工复核。

[现状事实] 线上 Demo 能展示视频、节点、情景、复述和报告，但进入练习会暂停视频，完成后会修改播放位置；全屏暗幕与 88%—94% 高底板明显遮挡视频；右侧作者头像位直接显示“财＋”。

# 2. 问题、用户与产品边界

## 2.1 用户问题

财经长视频的主要问题不是“信息不够”，而是用户容易记住结论却遗漏中间机制、成立条件、反向力量和不确定性。收藏、摘要和思维导图能帮助回忆，却不能证明用户能在条件变化时重新推演。

[产品决策] 产品不追求把视频改造成连续考试，而是在作者证据仍在屏幕和声音里时，让用户完成一个最小认知动作，并把动作留下的证据组合成过程总结。

## 2.2 目标用户

- 核心用户：18—35 岁、常看 3—20 分钟财经科普但容易“听懂即忘”的用户。
- 次级用户：商科学生、财经行业新人和希望检查逻辑链的内容消费者。
- 内容角色：产品、运营、财经审核员和测试人员。
- 非目标用户：寻求实时行情、具体资产推荐、仓位、目标价、收益承诺或自动交易的人。

## 2.3 核心用户故事

- 作为观看者，我希望问题出现时视频继续，以免知识卡打断作者的语气和证据。
- 作为入门用户，我希望有时只补一个背景或拨一个条件，而不是每次都做题。
- 作为容易套结论的用户，我希望看到同一政策在不同条件下为何出现竞争路径。
- 作为回访用户，我希望刷新或关闭后能恢复节点、答案和观看位置。
- 作为审核员，我希望每个概念、关系、问题和报告结论都能追到证据与审核版本。
- 作为测试人员，我希望任何模型或网络故障都不阻断最短学习闭环。

## 2.4 产品定义

> 财经推演室是一种嵌入财经长视频时间轴的轻量伴学体验。财包在经审核的关键时点发出短邀请；用户可忽略，也可在不停止视频的半屏容器里完成一个小动作；片尾再进入可选的条件沙盘、复述和证据报告。

## 2.5 P0 范围

P0 必须完成：

- 一个经授权、经人工审核、可公开演示的内容包。
- 真实媒体、字幕、作者头像、作者昵称、来源与时间码一致。
- 四次以内自动邀请和六类可渲染轻交互。
- 不停播、无蒙层、最高 48vh 的半屏容器。
- 版本化本地恢复和 Express 会话/事件接口。
- 片尾规则沙盘、反例深挖、文字复述与证据报告最短路径。
- AI 超时、无效 JSON、断网和会话丢失时的确定性兜底。
- 内容发布门禁、财经安全、埋点和端到端测试。

P0 明确排除：

- 任意 URL 自动抓取并发布视频。
- 未经创作者 OAuth 或授权声明的批量下载。
- 实时行情、具体涨跌幅、目标价、买卖、仓位与收益承诺。
- 独立 AI 聊天页、自由问答、Subagent、常驻悬浮球和整机光环。
- 创作者自助后台、社交排名、长期财富画像和风险偏好推断。
- 把同一套股票/黄金/汇率结论硬套到所有内容包。

## 2.6 成功定义

| 层级 | 成功表现 | 不可替代的证据 |
|---|---|---|
| 产品形态 | 用户知道财包与原作者不同，视频始终在继续 | 播放状态、作者元数据与 UI E2E |
| 学习动作 | 用户至少完成一次条件或因果动作 | `InteractionTrace` 事件 |
| 条件意识 | 用户不把政策变化直接等同资产必然方向 | 反例/复述人工评测 |
| 报告可信 | 每项已观察能力可展开到动作和视频证据 | `supportingEventIds` + `evidenceIds` |
| 系统可靠 | 模型失败仍能得到非空总结 | 故障注入 E2E |

# 3. 端到端产品流程

## 3.1 内容生产流程

1. 输入经授权的媒体文件或创作者 OAuth 可访问资产。
2. FFmpeg 读取时长、音轨、编码和指纹，生成 ASR 音频与 OCR 关键帧。
3. ASR 产出逐句时间轴；OCR 产出画面文字；多模态分析补充视觉语义。
4. 语义分析生成概念、背景、因果关系、条件、反例和证据引用候选。
5. 确定性 Planner 根据最短间隔、总预算、字幕密度和认知负荷选择自动邀请候选。
6. Payload Author 为可渲染种类生成有界文案；财经安全与 Schema 再校验。
7. CoverageReport 列出缺失证据、不可渲染种类、方向冲突和人工决策项。
8. 内容审核员复核授权、字幕、时间码、概念、因果边、选项、规则和边界文案。
9. 发布不可变 `ApprovedExperience`；客户端只消费已审核版本。

[产品决策] ASR/OCR 时间戳是候选定位依据，不是事实审核结论。模型不得自行把模糊画面或字幕补写成精确事实。

## 3.2 用户观看流程

```text
进入财经视频
  ↓
视频、音频、字幕与作者信息正常播放
  ↓
命中经审核时间窗 → 轻邀请出现 4—6 秒（视频继续）
  ├─ 忽略/关闭 → 收入时间轴，继续观看
  └─ 打开 → 底部半屏微交互（视频继续）
               ├─ 完成 → 记入证据，收起
               ├─ 跳过 → 记为未观察，收起
               └─ 用户主动暂停/续播 → 尊重用户播放控制
  ↓
后续节点按间隔、跳过与负荷降频
  ↓
视频结束 → 展示过程总结卡
  ├─ 直接看证据报告
  └─ 进入片尾深挖：完整沙盘 → 反例 → 复述 → 报告
```

## 3.3 时间预算

[产品决策] 4 分 16 秒视频本身已超过 4 分钟，不能再要求“自然看完 + 全部任务 ≤4 分钟”。V2.3 改为以下可测预算：

| 阶段 | 预算 |
|---|---:|
| 单次轻邀请可见 | 4—6 秒 |
| 单次观看中动作 P75 | ≤12 秒 |
| 单视频自动邀请 | ≤4 次 |
| 观看中新增操作总时长 P75 | ≤60 秒 |
| 片尾深挖最短路径 | ≤3 分钟 |
| 评审演示代表链路 | 允许审核时间轴跳转，≤4 分钟 |

# 4. 信息架构与入口

## 4.1 用户侧信息架构

```text
视频 Feed
├─ 视频、字幕、章节、作者和社交区
├─ 财包轻邀请（自动出现，短时可忽略）
├─ 节点轨道（只显示状态与可回看节点）
├─ 学习轨迹入口（恢复/片尾）
└─ 半屏微交互
   ├─ 背景补丁
   ├─ 快判断
   ├─ 单条件拨片
   ├─ 因果补边
   ├─ 轻反例
   └─ 概念对照

片尾深挖
├─ 逻辑地图
├─ 完整条件沙盘
├─ 反例挑战
├─ 三句话复述
└─ 证据型理解报告
```

## 4.2 入口裁决

[产品决策] 主要入口不是 POI 链接，也不是 AI Tab。POI 通常指向地点/商品或离开内容语境，本产品入口是绑定视频时间窗、证据和学习目标的 `CuePill`。

- 自动入口：关键时点出现的财包轻邀请。
- 恢复入口：视频下方“学习轨迹”小按钮，显示完成数和下一步。
- 回看入口：已出现节点的时间轴点；未出现节点不可伪造完成。
- 片尾入口：视频结束后的过程总结卡。
- 调试直达：仅开发/测试环境，生产默认关闭。

## 4.3 邀请外观与行为

- 位置：视频底部安全区上方，避开字幕、底部导航和系统手势区。
- 高度：建议 64—84px；触控目标 ≥44px。
- 内容：财包小头像、动作型标题、一句 20 字以内说明、进入箭头。
- 生命周期：淡入 150—220ms，显示 4—6 秒，未操作后收进节点轨道。
- 同屏：只允许一个邀请；不得叠加 Toast、评论面板或另一个 Sheet。
- 关闭：只关闭本次；连续两次关闭/跳过后后续自动邀请降级或停止。

# 5. 财包 Agent 产品规范

## 5.1 角色

[产品决策] 财包是“财经因果陪练”，不是内容作者、分析师、投顾或市场预测者。它帮助用户看见背景、机制、条件、反例与证据，不代替用户下结论。

## 5.2 出现位置

- 轻邀请左侧头像。
- 半屏标题和一句反馈。
- 复述评价的“已说出/还缺/可改写”提示。
- 报告中的下一步建议。

不得出现：

- 作者头像位、作者昵称、关注按钮或内容来源位置。
- 常驻悬浮球、整机黄边、连续动画或永久占用视频上方空间。
- 图谱、证据、沙盘结果的主体位置。
- 未经用户进入的独立聊天窗口。

## 5.3 人格与文案

- 简洁：先给一个动作，再给一句理由。
- 苏格拉底式：询问“还缺什么条件”，不宣布唯一答案。
- 条件化：使用“可能、支撑、压制、冲突、信息不足”。
- 证据优先：反馈引用当前视频证据或明确说明尚无证据。
- 不确定性诚实：不知道时返回“信息不足”，不补编。

禁止文案：`必涨`、`必跌`、`稳赚`、`抄底`、`买入`、`卖出`、`仓位`、`目标价`、`收益承诺`。

## 5.4 财包状态

| 状态 | 用户可见表现 | 系统行为 |
|---|---|---|
| `idle` | 无财包元素 | 正常播放 |
| `eligible` | 不立即展示 | 等待时间窗与频控 |
| `offered` | 轻邀请 4—6 秒 | 视频继续，记录曝光 |
| `expanded` | 半屏微交互 | 视频继续，记录进入 |
| `completed` | 一句反馈后收起 | 记录动作与证据 |
| `dismissed` | 收进时间轴 | 不判错、不打扰 |
| `deferred` | 片尾“待补” | 进入片尾任务池 |
| `reported` | 报告证据项 | 可回看原事件 |

# 6. 触点编排规则

## 6.1 自动选择标准

触点只有同时满足以下条件才可自动邀请：

- 当前片段出现新概念、关键背景、因果跳步、条件分叉、反例或易混关系。
- 用户可以在 12 秒内完成一个清晰动作。
- 证据窗口、学习目标和反馈均已审核。
- 不与密集字幕、作者关键结论、系统控件或上一触点冲突。
- 满足内容包最短间隔和单视频预算。

## 6.2 频率硬约束

| 约束 | P0 值 |
|---|---:|
| 自动邀请总数 | ≤4 |
| 内容时间轴节点总数 | ≤6 |
| 自动邀请最短间隔 | ≥45 秒 |
| 同时展开数 | 1 |
| 邀请可见时间 | 4—6 秒 |
| 预计动作时间 | ≤12 秒 |
| 半屏高度 | ≤48vh |
| 连续跳过降频 | 2 次后停止强邀请 |

[产品决策] 内容节点数和自动邀请数不是同一个指标。节点可以保留在时间轴或片尾，但只有符合频控的节点自动弹出。

## 6.3 首包候选节点裁决

PM 内容包的六个节点间隔为 27、53、64、36、50 秒，不能全部自动邀请。V2.3 推荐初始计划如下：

| 时间 | 原节点 | CueKind | 自动邀请 | 裁决理由 |
|---|---|---|---|---|
| 00:08 | 先认清资本在找什么 | `context_card` | 否 | 作为无强弹背景标签，避免与 00:35 太近 |
| 00:35 | 相关性还是因果 | `quick_judgment` | 是 | 第一个主动判断 |
| 01:28 | 拼上美元外溢一步 | `causal_stitch` | 是 | 与前点间隔 53 秒 |
| 02:32 | 换一个国家推演 | `condition_slider` | 是 | 与前点间隔 64 秒 |
| 03:08 | 利差交易反例 | `counterexample_flip` | 否 | 与 02:32 只有 36 秒，转片尾/主动回看 |
| 03:58 | 讲清资本洪流 | 片尾复述入口 | 是 | 与 02:32 间隔 86 秒，结束后进入复述 |

[假设] 经认知负荷测试后，03:58 可只显示片尾提醒而非题目；Planner 可进一步减少自动邀请，但不得超过四次。

## 6.4 快进、拖动与重复触发

- 正常播放进入时间窗：最多触发一次。
- 从窗前拖到窗后：若跨度 ≤30 秒且当前无面板，可显示一次弱邀请；否则仅标记 `missed`。
- 从后向前拖回：同会话不再自动强弹，可从时间轴主动进入。
- 刷新恢复：已 `offered` 的节点不再次自动弹；未出现节点按当前位置重新评估。
- 内容版本变化：旧会话保留只读历史，新版本创建新会话，不混合证据。

# 7. 观看中六类轻交互

## 7.1 模板总表

| CueKind | 用户动作 | 适用问题 | P0 反馈 |
|---|---|---|---|
| `context_card` | 点开/确认一个背景 | 缺少概念或制度背景 | 一句关键点 + 证据 |
| `quick_judgment` | 选择最合理判断或不确定 | 作者揭晓前的预测 | 合理处 + 缺失条件 |
| `condition_slider` | 拨一个变量 | 条件变化是否改写路径 | 激活路径 + 竞争力 |
| `causal_stitch` | 选择缺失的中间边 | 结论跳步 | 完整因果边 |
| `counterexample_flip` | 选择更强反向条件 | 绝对化结论 | 机制仍在/被盖过 |
| `concept_compare` | 对照两个易混概念 | 相关/因果等混淆 | 一句话关键区分 |

## 7.2 共同交互约束

- 打开前后 `video.paused` 状态不得被产品代码改变。
- 不自动调用 `video.currentTime = ...`；证据回看只能由用户主动发起。
- 面板最高 48vh，无全屏 shade、blur 或 pointer-blocking mask。
- 画面上部仍可触发用户自己的播放/暂停动作。
- 主动作、关闭与跳过触控目标均 ≥44×44px。
- 选择后先显示与当前证据有关的反馈，不立即全屏跳转。
- 完成和跳过都可收起；跳过不判错、不扣分。
- 所有模板必须有静态审核反馈，模型不可用仍能完成。

## 7.3 快判断选项规范

[产品决策] `quick_judgment` 使用 3 个有意义的业务选项 + “我不确定”；“先继续看”是独立跳过动作，不计入选项数组。

合格选项应代表不同机制或条件，不用同义改写制造伪选择。首次答案与最终答案分别记录；用户改选不覆盖首次思路。

示例：

```text
如果美国降息、而其他经济体利率暂时不变，美元一定走弱吗？
A. 相对回报下降，美元承压路径更强
B. 避险或增长差异可能抵消利差路径
C. 仅凭美国降息无法判断相对利差
D. 我不确定
[先继续看]
```

## 7.4 单条件拨片

观看中只能修改一个变量并比较两到三个状态，不能把四变量完整沙盘塞进半屏。反馈只回答“哪条路径被激活/压制”，不输出资产确定方向。

## 7.5 因果补边

- 前件、后件和候选中间边始终可见。
- 候选 2—3 个，其中错误项必须是有诊断价值的常见跳步。
- 正确反馈说明中间机制；错误反馈说明为何无法从前件直接跳到后件。
- 不把股票、黄金或汇率方向当作答对奖励。

## 7.6 轻反例与概念对照

- 轻反例只要求识别一个更强反向力；完整两阶段反例放片尾。
- 概念对照优先采用左右卡、滑动或一键揭示，不强行变成题目。
- 无需作答的背景卡仍可形成 `observed` 证据，但只能证明用户看过，不能证明掌握。

# 8. 半屏容器与播放不变量

## 8.1 尺寸和层级

- Sheet 高度：内容自适应，`max-height: 48vh`，大屏手机可再限制为 420px。
- 顶部圆角：20—28px；拖拽条为辅助，不是唯一关闭方式。
- 不渲染全屏 `.shade`；允许面板自身投影区分层级。
- 390×844、393×852、430×932 下至少保留 52% 视频画面。
- 横屏和桌面容器以不遮挡播放控制为原则，可降为右侧窄面板，但 P0 只承诺可操作。

## 8.2 播放不变量

```text
打开邀请前的 paused/muted/currentTime
  = 打开后产品主动修改前的 paused/muted/currentTime

完成、跳过、关闭
  不调用 play()、pause()、seek() 或修改 muted
```

用户在面板打开时主动点击视频暂停是有效用户操作；关闭面板后保持用户选择，不强制恢复。

## 8.3 冲突处理

- 评论、分享、作者主页或系统权限弹窗打开时不展示新邀请。
- 字幕位于底部时，邀请向上避让或降级到轨道点。
- iOS 安全区通过 `env(safe-area-inset-bottom)` 处理。
- 软键盘打开后复述任务采用片尾页面，不在播放中半屏强撑。

# 9. 片尾深挖

## 9.1 进入方式

视频自然结束后展示非阻断的过程总结卡：完成了什么、还有什么未观察、可直接看报告或继续深挖。用户退出时基础报告仍可生成，不强迫完成所有环节。

## 9.2 逻辑地图

- 主体是审核概念、因果边、条件和证据，不以财包占据中心。
- 默认只展开与本次事件有关的路径。
- 点击节点/边可查看证据窗口；未审核边对用户不可见。
- 不显示与当前内容包无关的股票/黄金/汇率路径。

## 9.3 完整条件沙盘

四个主变量为利率、通胀、增长、避险情绪。相对政策差异以场景条件标签表达，不伪装成第五个自由变量。场景 A/B/C 是预设入口，用户载入后仍须点击“运行这组条件”。

结果顺序固定：

1. 展示被激活的路径与 `evidenceIds`。
2. 展示竞争路径与条件标签。
3. 输出枚举 `support_dominant | pressure_dominant | conflict | insufficient`。
4. 展示边界和“知识情景推演，不构成投资建议或市场预测”。

![图 4｜PM 情景页实测：国家条件叙事可复用；V2.3 改为先修改条件、再点击运行，且不暂停视频|w=210](assets/prd-v2.3/moneybaby-online-simulate.png)

## 9.4 反例深挖

第一步判断反常结果是否推翻原机制；第二步补充可能盖过原机制的条件。合理条件必须来自审核内容或规则表。跳过后报告显示“边界意识尚未观察”，不显示错误分数。

## 9.5 三句话复述

任务模板：

1. 发生了什么政策或背景变化。
2. 通过什么中间机制影响了哪条路径。
3. 在什么条件下结论可能改变或冲突。

P0 文本 18—300 字。Web Speech Recognition 仅作 feature-detect 的输入增强；不支持、权限拒绝或识别失败时回退文字。客户端与服务端均不保存原始语音。

# 10. 证据型理解报告

## 10.1 报告定位

> 这不是成绩单，而是用户本次如何思考、哪些机制被观察、哪些条件尚未验证的记录。

## 10.2 报告模块

- 本期内容与内容包版本。
- 已观察动作：题目、首次回答、最终回答、沙盘输入、复述命中。
- 掌握候选：必须有至少一条行为证据与一条视频证据。
- 尚未观察：跳过、未出现或证据不足的机制。
- 条件意识：是否提到条件、竞争路径和不确定性。
- 错误模式：绝对化、机制缺失、条件缺失、概念混淆。
- 前后复述对比（若有二次提交）。
- 定向回看片段、迁移题和下一步。
- 免责声明与内容审核版本。

## 10.3 归因约束

- 每个“已观察/掌握候选”必须有 `supportingEventIds` 和 `evidenceIds`。
- 仅曝光背景卡只能记为“看过”，不能记为“掌握”。
- 已跳过节点不得生成能力印章。
- 证据不足显示“尚未观察”，不显示低分、红叉或羞辱性文案。
- 报告不得因模型超时为空；确定性 Builder 先生成，模型最多润色。

# 11. 首个内容包决策

## 11.1 内容主题变化

[现状事实] V2.0 固定主题为“美联储降息如何影响股票、黄金和汇率”。PM 真实视频候选实际聚焦“美元降息、全球资本流动、本国周期、相对利差和套息交易”。两者相关但不是同一内容覆盖。

[产品决策] V2.3 改为**内容包定义学习目标**。首包候选 `fed-rate-global-capital-001` 只对它真正覆盖的机制负责；股票和黄金路径进入后续内容包，除非审核证据证明本视频确实覆盖。

## 11.2 首包候选学习目标

- 区分“美国降息”与“全球同步降息”。
- 理解美元资产相对回报变化如何影响跨境资本搜索。
- 识别本国通胀与增长周期对政策空间的约束。
- 识别相对利差、汇率与套息交易路径。
- 能用至少一个反向因素说明政策不必同步、资本传导也可能被改写。

## 11.3 首包候选发布门禁

[发布阻塞项] 以下任一缺失均保持 `draft`：

- 视频和创作者授权证明、分发范围、期限与下架机制。
- 视频 SHA-256、时长、编码、字幕版本和作者元数据一致。
- 最终 ASR、OCR、章节、六个证据窗口由人工复核。
- 概念、因果边、条件、反例和选项均有审核人、审核时间和版本。
- 自动邀请计划满足最短 45 秒、最多 4 次。
- 所有公开文案通过财经安全检查。

## 11.4 作者与来源完整性

[产品决策] 内容包必须携带并渲染 `authorId`、`authorName`、`authorAvatarUrl`、`sourceUrl`、`licenseStatus`。媒体与作者元数据不匹配时，构建或发布检查必须失败，避免当前 GitHub Pages 上财经视频显示 `@李子柒` 的问题。

# 12. AI / ASR / OCR 技术路径

## 12.1 当前已实现骨架

[现状事实] 服务端已有以下分阶段能力：媒体预处理、ASR/OCR 适配、语义图分析、有界抽取、可选批评、最多两次修复、确定性评分与 Planner、方向规则、payload 成稿、DraftExperience 和 CoverageReport。

[现状事实] Provider 配置支持 OpenAI 兼容的 Minimax、火山方舟豆包、火山大模型 ASR 与可选火山 OCR。真实密钥、模型权限、成本、延迟和输出质量仍需在受控环境验收。

## 12.2 模型分工

| 能力 | 模型可做 | 模型不可做 |
|---|---|---|
| ASR/OCR | 提供带置信度的文字候选 | 把低置信度候选直接发布 |
| 多模态理解 | 识别概念、背景、关系和候选证据 | 编造视频中不存在的事实/时间码 |
| 触点生成 | 在 Schema 内生成候选文案与选项 | 绕过频控、审核或可渲染白名单 |
| 财经方向 | 提取信号供规则使用 | 修改规则锁定的资产方向 |
| 复述评价 | 按 rubric 标注错误模式与证据引用 | 给投资建议、人格/财富画像 |
| 报告 | 润色确定性证据摘要 | 新增没有事件支持的能力结论 |

## 12.3 时间戳权威

- 媒体时长来自 FFprobe。
- 逐句语音时间戳来自 ASR 原始结果。
- OCR 时间来自抽帧位置。
- 模型只能引用给定 `evidenceIds`，不得直接输出自由时间戳。
- 人审确认后才进入 `ApprovedExperience`。

## 12.4 有界修复与故障

- Schema 或安全校验失败最多修复 2 次。
- 仍失败的候选进入 CoverageReport，不进入运行内容。
- 模型超时、网络错误或无效 JSON 不阻塞其他候选和人工成稿。
- 复述实时评价 P95 超过 8 秒立即使用模板反馈。
- 规则未知组合一律 `insufficient`，不得请求模型猜方向。

# 13. 技术架构

## 13.1 架构决定

[产品决策] P0 继续使用 Vue 3/Vite/Pinia/Express TypeScript 单体式代码仓，不引入 moneybaby 的 React/Vinext 前端，也不恢复旧 FastAPI 双栈。

![图 6｜V2.3 参考架构：模型生成候选、人审发布；客户端只运行审核计划](assets/prd-v2.3/03-v23-architecture.png)

## 13.2 两条数据流

内容生产流：

```text
Authorized Media
→ FFmpeg / ASR / OCR / Keyframes
→ SemanticTimeline
→ Candidate + Critique + Repair
→ Deterministic Planner + Direction Rules
→ Authored Payload
→ DraftExperience + CoverageReport
→ Human Review
→ ApprovedExperience
```

用户运行流：

```text
ApprovedExperience
→ Player + CueOrchestrator
→ CuePill + HalfSheet Renderer
→ InteractionTrace + localStorage mirror
→ Session/Event API
→ Simulation / Retell Evaluation
→ Deterministic EvidenceReport
```

## 13.3 P0 会话策略

- Express 进程内存态保存会话和事件，浏览器 `localStorage` 保存同版本镜像。
- 恢复顺序：先请求服务端；不存在时用本地快照补回；两者冲突以不可变事件时间与幂等键合并。
- 本地 key 必须包含 `videoId + contentVersion`。
- 保存播放器 `currentTimeMs`、已曝光/进入/完成/跳过、首次/最终答案、沙盘输入、复述文本引用和最后更新时间。
- P0 不承诺多实例一致性；生产化迁移 PostgreSQL + 持久队列。

## 13.4 部署口径

[现状事实] GitHub Pages 只承载静态前端评审版，不能证明 Express、ASR/OCR 或模型服务已生产部署。

[产品决策] P0 集成验收需要一个公网 HTTPS 服务端，域名、CORS、密钥隔离、日志和健康检查全部可验证。具体平台由研发评审选择，不把单一平台写成产品依赖。

```text
ApprovedExperience    审核后不可变运行内容
VideoContext          视频、作者、字幕、时间和证据上下文
CoachSession          版本化会话与当前流程状态
InteractionTrace      曝光、进入、完成、跳过、回看与答案
SimulationInput       四变量、场景标签、内容/规则版本
SimulationResult      路径、证据、方向枚举与边界
RetellEvaluation      rubric 标签、引用和兜底模式
EvidenceReport        仅由事件和证据生成的过程总结
```

## 14.4 SimulationResult

```json
{
  "ruleVersion": "finance-causal-v1",
  "direction": "conflict",
  "activatedPaths": ["valuation_support", "earnings_pressure"],
  "evidenceIds": ["e-rate", "e-growth"],
  "conditions": ["rate_down", "growth_recession"],
  "boundary": "两条路径同时成立，当前不能给出单一方向。"
}
```

枚举固定为 `support_dominant | pressure_dominant | conflict | insufficient`。未知、空输入或规则未覆盖组合返回 `insufficient` 并带原因。

## 14.5 事件幂等

事件键建议为 `sessionId + triggerId + action + attemptNo`。服务端返回接收游标；客户端重试不得产生重复完成、重复掌握或重复指标。

# 15. 内容来源与抖音边界

## 15.1 合法来源优先级

1. 项目方持有或已获授权的本地媒体上传。
2. 创作者通过抖音开放平台 OAuth 授权的账号与视频。
3. 官方 iframe/分享能力允许的公开播放形态。
4. 公开页只做元数据能力探测，不作为批量下载和再分发授权。

[产品决策] 开源下载器只可在明确授权、内部测试和法律允许的范围内作为工具评估，不进入 P0 的默认产品链路，也不绕过登录、验证码、签名或访问控制。

## 15.2 来源数据要求

每个媒体资产必须记录：来源 URL、作者 ID/昵称/头像、取得方式、授权声明、允许用途、授权期限、文件指纹、取得时间、审核人和下架状态。

## 15.3 失败行为

- 只有公开元数据、无媒体：返回 `metadata_only`，不可创建分析 Job。
- 没有 rights attestation：返回 403，不能进入 ASR/OCR。
- 视频下架或授权过期：停止新会话；历史报告保留版本引用但不继续分发媒体。

# 17. 约束、安全、隐私与非功能需求

## 17.1 财经安全

- 对“买什么、仓位、目标价、稳赚、什么时候卖”等请求固定拒绝交易建议，并转回机制解释。
- 结果不含具体价格、涨跌幅、收益概率和操作动作。
- 免责声明在沙盘结果、复述反馈与报告均可见。
- 模型输出经过 Schema、禁词、证据引用和规则方向校验。

## 17.2 隐私

- 不保存原始语音；浏览器识别结果只按复述文本策略处理。
- 复述文本最小化保存，用户可删除；日志只记长度、模式和错误标签。
- 不推断财富、收入、持仓、风险偏好、政治倾向或心理特征。
- 密钥仅在服务端环境变量中，前端包和健康接口不得返回明文。

## 17.3 性能

| 能力 | 目标 |
|---|---:|
| 已缓存内容包响应 | ≤300ms |
| 半屏打开反馈 | ≤100ms |
| 规则沙盘结果 | ≤500ms |
| 复述评价 P95 | ≤8 秒，超时立即模板兜底 |
| 视频拖动误差 | ≤1 秒 |
| 邀请动画 | 60fps 目标，不阻塞播放 |

## 17.4 兼容与可访问性

- 视口：390×844、393×852、430×932 和桌面手机容器。
- iOS Safari、Android Chrome 最近两个大版本；桌面 Chrome/Edge/Safari 最近两个大版本。
- 触控目标 ≥44px；键盘可达；图标有可访问名称；状态不只靠颜色。
- 支持 `prefers-reduced-motion`；动效不影响作答。
- 字幕、财包邀请和面板文案在 200% 字体缩放下仍可阅读或滚动。

## 17.5 故障兜底

| 故障 | 用户结果 |
|---|---|
| 视频失败 | 封面 + 文字证据 + 片尾深挖入口 |
| 内容包失败 | 普通视频继续，不展示未经审核触点 |
| 模型超时/无效 JSON | 静态审核反馈，流程可完成 |
| 规则未知组合 | `insufficient`，说明缺少条件 |
| 断网 | 本地会话继续；恢复后幂等上报 |
| 服务端会话丢失 | localStorage 快照补回 |
| 语音不支持/拒绝 | 文字输入，不阻塞复述 |
| 证据片段不可用 | 显示审核文字摘要和来源状态 |

- 每条标签引用原文片段或证据；模型无效时模板反馈可完成。
- 浏览器不支持语音、权限拒绝、识别中断和空识别均回退文字。

## 18.10 故障与恢复 E2E

- 模型超时、无效 JSON、断网、刷新和服务端会话丢失后仍可完成最短路径。
- 报告不可为空；至少显示已观察、尚未观察和下一步。
- 幂等重试不重复计数。
- 内容包更新不污染旧版本证据。

- 取消“AI解析 Tab + 一次轻提示”，改为时间轴多次轻邀请 + 学习轨迹恢复入口。
- 明确播放中任何交互都不自动暂停，不是“主动进入后暂停”。
- 自动邀请从最多 6 次收紧为最多 4 次；内容节点仍可最多 6 个。
- 内容目标从固定股票/黄金/汇率改为由审核内容包定义。
- 首包候选改为 PM 的美元外溢/全球资本主题，但保持 draft 和授权阻塞。
- 技术基座以当前 Vue/Vite/Express 为准，不再写 React。
- 合并已实现多阶段生成管线和真实 API As-Is。

## 23.2 相对 PM V2.2

- 删除进入练习自动暂停、完成后自动 seek、全屏暗幕和 88%—94% 底板。
- 财包不替换作者头像，也不使用常驻整机黄边。
- 二选一升级为 3 个有意义选项 + 不确定 + 独立跳过。
- 观看中只做单变量拨片；完整沙盘必须点击运行并放片尾。
- 报告能力结论必须绑定事件与证据，删除静态印章。
- 把 PM React 原型定位为内容/UI 参考，而非实施基座。
- 修正“完整自然流程 ≤4 分钟”的不可能约束。

## 23.3 文档验收

- 目录、图号、链接、事实标签、需求追溯和来源冲突已核对。
- 所有产品图标明“目标形态”或“现状证据”，不把 PM 原型冒充最终 UI。
- Markdown 与 PDF 内容一致。
- PDF 逐页渲染为 PNG，检查中文字体、乱码、溢出、重叠、表格拆页和图片清晰度。
- 本 PRD 评审通过后更新 `AGENTS.md` 与 `docs/AGENT_HANDOFF.md` 的权威文档指针。

# 附录 B. 验收示例

## B.1 Given / When / Then：不停播

Given 视频正在播放且声音开启  
When 到达 01:28，用户打开因果补边并完成  
Then 视频持续播放，muted 不变，currentTime 单调前进，面板最高 48vh，无全屏蒙层，完成后不发生自动 seek。

## B.2 Given / When / Then：证据报告

Given 用户只完成 01:28 因果补边，跳过 02:32，未做复述  
When 打开报告  
Then 报告只把因果补边记为已观察；国家条件和复述显示尚未观察；每项都可追到事件与证据；页面不显示总分或静态能力印章。

## B.3 Given / When / Then：模型故障

Given 复述评价模型超时且服务端会话刚丢失  
When 客户端从 localStorage 恢复并提交复述  
Then 服务端幂等补回事件，8 秒内进入模板反馈，用户仍能打开非空报告，且报告明确标注基础反馈模式。

## B.4 Given / When / Then：内容门禁

Given PM 内容包完成 Schema 适配但授权缺失、概念仍未审核  
When 尝试发布  
Then 发布接口拒绝，状态保持 draft，CoverageReport 列出授权和审核缺口，公开客户端无法获取该版本。
