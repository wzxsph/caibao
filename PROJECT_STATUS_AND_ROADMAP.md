## 1. 项目目标

财经推演室面向观看财经长视频的用户。核心问题不是“用户有没有看到结论”，而是“用户是否理解结论背后的因果路径、成立条件和例外”。

产品闭环：

> 视频建立语境 → 财包在关键节点邀请帮练 → 用户预测与推演 → 反例校准 → 主动复述 → 图解式个人推演复盘

项目不提供行情预测、买卖建议、仓位建议或收益承诺。所有结果必须使用“可能、支撑、承压、冲突、信息不足”等条件化表达。

## 2. 当前完成进度

### 2.1 已完成产品能力

- 抖音式 375×812 竖屏 Feed 场景。
- 模拟视频播放、暂停、字幕、章节、进度与关键节点。
- AI 帮练不自动打断：节点到达后仅出现底部邀请卡，用户主动选择是否开始。
- 六类嵌入式帮练：
  1. 名词预检；
  2. 预测下一步；
  3. 因果链补全；
  4. 变量推演；
  5. 反例挑战；
  6. 主动复述。
- 选择题统一为两个选项，减少阅读和决策负担。
- 变量沙盘覆盖增长、避险情绪以及股票、黄金、美元的竞争路径。
- 最终报告升级为图解式“个人推演复盘”，包含中心变量、三条主路径、反向力量、能力印章、回看与迁移挑战。
- 支持报告直达演示：`/?view=report`。
- 本地静态数据兜底，无模型与网络时仍可跑通 Demo。

### 2.2 财包 Agent

财包是财经推演室的 AI 帮练助手与统一 IP。

- 形象：佩戴耳机和金色铃铛的白色小狗。
- 角色：用户的“财经推演搭子”，不是权威老师、分析师或投资顾问。
- 启动方式：用户点击帮练邀请卡中的“叫醒财包”。
- 当前表现：进入帮练后，财包以头像和对话气泡提出问题；每个环节由财包持续陪伴。
- 当前资产：`public/caibao.jpg`。

## 3. 产品规范

### 3.1 AI 帮练触发规范

- 禁止视频一开始集中展示全部练习。
- 禁止关键节点自动弹出全屏题目或自动暂停。
- 到达节点后，在视频下方出现轻量邀请卡；视频保持播放。
- 用户点击“叫醒财包”后，视频暂停并进入练习。
- 用户关闭邀请卡后继续观看，不重复强提示。
- 两个练习节点之间应至少留出足够观看时间，避免连续打断。
- 连续跳过后，后续邀请应降低视觉强度或减少频率。

### 3.2 题目规范

- 选择题只提供两个选项。
- 题目应让用户代入现实角色或亲身场景，例如公司老板、基金经理、普通消费者或向朋友解释。
- 优先考察“下一步会发生什么”“缺了哪个中间机制”“什么条件会使结论反转”。
- 禁止纯记忆定义题、照抄字幕题和无情境的教材式问法。
- 反馈顺序：用户判断 → 合理部分 → 缺失条件 → 原视频证据 → 条件变化。
- 不只判对错；必须说明为什么、何时成立、什么力量会打断。

### 3.3 报告规范

- 报告不是成绩单，不以单一分数作为核心结论。
- 优先使用思维导图、传导路径、竞争力量、能力印章等图解表达。
- 所有报告结论必须来自用户真实作答或操作证据。
- 报告必须包含下一步行动：精准回看、迁移挑战或复习任务。
- 固定展示“知识情景推演，不构成投资建议或市场预测”。

### 3.4 财包语言规范

- 自称“财包”，称用户为“你”，语气像聪明、耐心的搭子。
- 句子短，先说人话，再补财经术语。
- 推荐句式：
  - “先别背答案，我们看看中间发生了什么。”
  - “这条路说得通，但还有一股反向力量。”
  - “你抓住了方向，再补一个成立条件就完整了。”
- 禁止句式：
  - “根据我的预测，你应该买入……”
  - “这个资产一定上涨/下跌。”
  - “正确答案就是……”
- 财包不冒充持牌金融从业者，不输出具体交易动作。

## 4. 视觉与 IP 规范

### 4.1 当前视觉语言

- 主背景：近黑色与炭灰色，保持视频沉浸感。
- 主强调色：古金色，用于核心节点、按钮和财包铃铛意象。
- 正向/结构路径：青绿色。
- 信息/相对路径：低饱和蓝色。
- 反向力量与风险条件：克制的橙色。
- 卡片使用圆角、细描边和轻量光晕，避免传统课程后台感。

### 4.2 财包 IP 后续统一项

- 输出标准三视图、表情包与动作库。
- 建立统一头像尺寸：邀请卡、题目气泡、反馈卡、报告角标。
- 建立核心状态：发现知识点、思考、鼓励、提醒条件、发现反例、完成复盘。
- 统一耳机、铃铛、毛色、线条粗细和黄色背景的品牌规范。
- 后续需取得原创权或明确商用授权，并保存源文件与授权记录。
- 建议最终使用透明背景 PNG/WebP 和可编辑矢量母版；当前 JPG 仅作为原型资产。

## 5. 后续路线图

### 5.1 UI 改进

优先级 P0：

- 为财包增加首次启动动画和一句自我介绍，但总时长不超过 1.5 秒。
- 将题目反馈也改成财包对话，而不只是色块提示。
- 增加“本次不再提示”和连续跳过后的降频策略。
- 完善报告中节点点击、路径展开和证据回看。
- 完成 375×812、390×844、430×932 三档真机适配。

优先级 P1：

- 财包表情和动作随答题状态变化。
- 增加轻触震动、路径点亮和无障碍减弱动画选项。
- 建立组件规范：邀请卡、财包气泡、二选一、反馈、路径节点、印章、迁移卡。
- 设计空状态、加载、模型超时、断网与视频加载失败状态。

### 5.2 后端实现

优先级 P0：

- 把页面内 mock 内容拆成结构化 `videoContentPackage` JSON。
- 建立视频、章节、知识点、因果边、练习、证据时间戳和用户事件的数据模型。
- 建立会话状态与进度保存，刷新后恢复当前视频与已完成练习。
- 抽象 AI 服务层：`generateQuestion`、`evaluateAnswer`、`evaluateRetell`、`generateReport`。
- 保留规则引擎作为方向判断来源；模型只负责解释与语言生成。
- 所有模型调用配置超时、重试、缓存和静态兜底。

优先级 P1：

- 接入字幕解析、章节切分和知识图谱候选生成。
- 建立人工审核台，审核核心概念、因果边、题目、反馈和时间戳。
- 接入语音复述与低置信词确认。
- 增加事件分析：邀请曝光、开始率、跳过率、完成率、错误类型、回看与迁移表现。
- 增加内容包版本、审核状态、发布与回滚能力。

### 5.3 AI Agent 实现

- Main Agent 读取当前字幕、章节、用户历史错误和已掌握节点。
- 财包根据上下文决定当前动作：轻提示、提问、追问条件、解释、推荐回看或进入迁移题。
- 工具/能力分层：
  - 视频证据检索；
  - 财经因果推演规则；
  - 复述结构化评价；
  - 学习状态更新；
  - 报告生成。
- 核心约束：模型不能自由决定资产涨跌方向，必须使用审核规则输出。
- 所有生成内容必须带视频证据或明确标记“当前视频未覆盖”。

## 6. 建议数据结构

```ts
type VideoContentPackage = {
  video: { id: string; title: string; duration: number };
  chapters: { id: string; start: number; end: number; title: string }[];
  concepts: { id: string; name: string; definition: string; evidenceAt: number }[];
  causalEdges: {
    from: string;
    to: string;
    conditions: string[];
    blockers: string[];
    evidenceAt: number;
    reviewed: boolean;
  }[];
  coachCues: {
    at: number;
    type: "concept" | "predict" | "chain" | "simulate" | "counter" | "retell";
    questionId: string;
  }[];
};
```
