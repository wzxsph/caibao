# 《财经推演室》Demo与路演脚本

## 1. 现场演示目标

在3—5分钟内让评委清楚看到三个变化：

1. 原视频的线性内容被重构为有条件、可追溯的因果模型；
2. 用户不是领取答案，而是通过预测、调参、反例和复述暴露并修正认知；
3. 系统不做行情预测或投资建议，只帮助理解机制和边界。

建议总时长4分20秒，演示者1人操作，另一人可负责口播或故障切换。

## 2. 3—5分钟逐步演示脚本

## 3. 演示输入与系统标准反馈

### 3.1 预测题

题目：美联储因经济快速恶化而宣布降息。股市下一步一定上涨吗？

演示选择：`一定上涨，因为融资成本下降。`

系统反馈：

> 你识别到了“利率下降→融资成本/折现率下降→估值可能获支撑”这条路径。但“一定上涨”忽略了降息原因：若经济快速恶化，企业盈利预期下降和风险溢价上升可能形成更强的反向力量。更准确的表达是：降息为估值提供支撑，但股市最终方向取决于政策缓冲与衰退冲击谁更强。

### 3.2 沙盘演示情景B

- 美联储利率：下降
- 通胀：下降
- 经济增长：衰退
- 避险情绪：高

标准输出：

- 股票：短期可能承压；利率路径为正，盈利与风险溢价路径为负，方向冲突。
- 黄金：通常可能获得避险和实际利率渠道支撑，但极端流动性抛售可能带来短期波动。
- 美元：降息压低收益率与全球避险美元需求方向相反，信息不足时标记“方向冲突”。
- 主导候选：若衰退冲击强，盈利下修可能暂时占主导。
- 边界：知识情景推演，不构成投资建议或市场预测。

### 3.3 主动复述

演示初答：

> 美联储降息以后美元一定会跌。大家就会去买黄金，所以黄金一定会上涨。

系统反馈重点：

- 有结论，但黄金机会成本这一中间机制缺失；
- 混淆美元汇率渠道与实际利率渠道；
- 两处把“可能”表述为“必然”；
- 未提其他经济体政策、通胀和避险情绪。

演示改写：

> 美联储降息通常会降低美元生息资产的回报；如果通胀没有下降得更快，实际利率也可能下降。黄金本身不生息，因此持有黄金的相对机会成本可能降低，从而给需求和价格带来支撑。但美元相对强弱、避险情绪和市场预期仍可能改变最终结果。

## 4. 30秒项目介绍

财经视频最危险的不是看不懂，而是以为自己看懂了。用户看完“降息利好股市和黄金”，往往只记住结论，忘了融资成本、实际利率、盈利预期和市场预期这些条件。财经推演室用AI把长视频重构成可运行的因果沙盘：用户先预测、再调变量、挑战反例，最后用自己的话讲出来。我们不替用户总结，而是让用户亲自运行一次视频里的逻辑。

## 5. 1分钟项目介绍

抖音精选有大量高质量财经长视频，但今天的消费终点仍然是“看完、收藏、继续刷”。以美联储降息为例，用户很容易记住“利好股市和黄金”，却不知道结论通过什么机制成立，也不知道衰退、通胀、避险情绪和其他经济体政策会让结果改变。

财经推演室先用AI把原视频拆成带时间戳的概念、因果链、条件和例外；观看中在关键点让用户先预测；看完后用户调整四个变量，系统展示股票、黄金和美元背后的竞争路径；再通过反例和三句话复述定位“把可能当必然”等误区。最终给出的不是一个分数，而是一份带回看片段和迁移题的理解报告。它同时完成信息层、形态层和功能层重构，让用户不是收到答案，而是真正学会运行一套逻辑。

## 6. 演示者记忆锚点

只需记住五句话：

1. **问题**：看懂每句话，不等于会运行整条逻辑。
2. **区别**：摘要替用户总结，我们让用户先判断并改变条件。
3. **创新**：每条因果边都有条件、打断因素和原视频证据。
4. **可信**：规则负责推演，AI负责解释；冲突和未知都明确展示。
5. **结果**：不是流程完成率，而是用户能否复述、处理反例并迁移。

## 7. 现场故障预案

| 故障 | 处理动作 | 口播 |
|---|---|---|
| 视频加载慢 | 直接点“跳到预测点” | “为了节省现场时间，我们从关键节点进入。” |
| 模型反馈超时 | 自动展示预置结构化反馈 | “固定视频的关键知识已审核，生成失败不会阻塞学习。” |
| 沙盘无结果 | 载入预设情景B | “MVP优先保证三种代表性经济情景的可靠解释。” |
| 语音识别失败 | 使用预填文字 | “产品同时支持文字复述，核心是评价用户自己的表达。” |
| 网络完全中断 | 切本地静态数据或录屏 | “推演计算由本地规则完成，不依赖实时行情。” |

## 9. 按官方评分权重强化的演示提示

| 时间点 | 评审维度 | 演示者必须说清的证据 |
|---|---|---|
| 开场0:00—0:30 | 场景洞察15% | “用户的问题不是没看见结论，而是不知道结论何时成立。” |
| 因果链与沙盘 | AI与产品结合20% | 明确说明“规则运行路径，AI结合原视频证据解释”，并展示一次节点回看 |
| 全流程 | 体验完整性15% | 预测、推演、反例、复述、报告不中断完成 |
| 前后回答对比 | 用户价值感30% | 展示用户从“降息黄金一定涨”修正为带中间因果、条件和不确定性的表达 |
| Skill页或架构页 | 创新与延展20% | 展示`finance_causal_simulation`的输入输出，并说明换知识图谱可迁移到其他财经主题 |

收尾建议：不要用“我们做了一个更丰富的视频总结”收尾，而要说：“原视频替用户给出结论，财经推演室让用户亲自运行、质疑并复述这套逻辑；被重构的不只是视频形态，而是用户使用财经知识的方式。”

| 追问 | 20秒回答 |
|---|---|
| 和AI总结有何本质不同？ | 总结把信息压短，用户仍被动接收；我们把内容变成可操作的条件化因果模型，并要求用户用预测、调参和复述证明理解。 |
| 沙盘是否过度简化经济？ | 它不模拟真实价格，只运行视频明确讲到的局部机制。每条边显示条件和打断因素，多条力量冲突时明确“不确定”。 |
| 为什么必须用AI？ | AI负责从跨时间段视频中抽取机制、理解自由复述并生成个性反馈；稳定推演仍交给规则，二者各做擅长的部分。 |
| 幻觉怎么控制？ | 固定视频离线解析、人工审核核心图谱、所有结论锚定时间戳、模型不能自由决定资产方向。 |
| 如何证明用户学会？ | 与视频+摘要做对照，测因果复述、条件识别、新情景迁移和一周后延迟回忆，而不只看完成率。 |
| 会不会成为投资建议？ | 不接实时行情，不给收益率、目标价或交易动作；只讲机制，使用审慎语言，并对交易请求明确拒绝。 |
