# 《财经推演室》官方规则映射与提交清单

更新时间：2026-07-21

## 1. 已确认官方信息

- 探索方向：内容理解、内容表达、内容使用。
- 技术思路：Main Agent依据用户请求、视频内容与上下文，动态编排Skill、Tool和Sub-agent。
- 核心要求：从抖音真实用户未满足需求出发，利用AI把内容价值发挥到极致。
- 交付原则：提交可运行作品，而非创意概念；形态可为Agent、App、Web或小程序，但必须可交互体验。
- 交付物：完整项目文档、可体验交互产品、代表性核心能力的Skill/Agent封装。

## 2. 评分映射

| 维度 | 权重 | 核心检查问题 | 项目证据 |
|---|---:|---|---|
| 场景与问题洞察 | 15% | 是否聚焦真实、具体、未满足的抖音需求 | 财经长视频“记结论、不懂路径与条件”的问题链 |
| AI能力与产品结合 | 20% | AI是否改变内容使用方式，链路是否真实可运行 | 多模态预解析、证据对齐、推演Skill、复述评价 |
| 体验完整性 | 15% | 是否形成可广泛体验的基本闭环 | 预测→推演→反例→复述→报告 |
| 用户价值感 | 30% | 是否提高学习、判断或行动效率并带来真实获得感 | 前后复述对比、迁移题、误区修正与回看片段 |
| 创新与延展潜力 | 20% | 是否有新消费方式及邻近场景迁移能力 | “运行逻辑”而非“阅读摘要”；Skill替换知识图谱后复用 |

## 3. 与官方案例的差异化

官方“教育视频→答题巩固”案例已经包含围绕视频知识点出题。因此本项目不能把预测暂停题当作核心创新。预测题只用于暴露用户认知；真正的产品核心是：多变量共同作用下的因果路径运行、方向冲突解释、反例校正、主动复述和新情景迁移。

## 4. 提交前硬性清单

- [ ] 固定视频可正常播放或使用合规片段演示。
- [ ] P0流程能在5分钟内完整跑通。
- [ ] 3个预测暂停点至少有1个在Demo中真实触发。
- [ ] A/B/C三组沙盘情景能稳定返回不同路径。
- [ ] 每条核心结论可回溯到视频时间戳。
- [ ] 冲突情景允许输出“方向不明”。
- [ ] 反例挑战能识别“忽略条件”和“把可能当必然”。
- [ ] 主动复述反馈包含中间因果、条件、不确定性与概念错误。
- [ ] 报告展示重构前后理解变化及迁移题。
- [ ] `finance_causal_simulation`有独立输入输出Schema并可重复调用。
- [ ] 页面固定展示“情景推演，不构成投资建议”。
- [ ] 断网、模型超时或生成失败时有预置结果兜底。
- [ ] 完整项目文档、可运行链接/安装包、Skill/Agent材料均已打包。

## 5. 当前仍缺失

- 比赛正式提交入口、截止时间与文件格式限制。
- 是否要求部署在指定平台，以及Skill/Agent的指定封装规范。
- 可使用的抖音内部能力、模型、视频解析接口和鉴权方式。
- 指定视频原文件、字幕、准确时间戳和创作者使用授权。
- 团队人数、实际开发天数与成员技术分工。

以上信息未补齐前，PRD继续采用“固定视频离线预处理 + 人工审核知识图谱 + 规则沙盘 + 受限AI解释”的黑客松假设。
