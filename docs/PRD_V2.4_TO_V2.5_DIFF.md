# 财经推演室｜PRD V2.4 → V2.5 差异与追溯

状态：Review Candidate 配套差异表  
日期：2026-07-23  
历史候选：`../财经推演室_PRD_V2.4.md`  
最新候选：`../财经推演室_PRD_V2.5.md`

## 1. 变更原因

用户于 2026-07-23 直接裁决：“交互时要自动暂停，进入财包”。这改变了播放状态机、事件记录和验收标准，属于规范行为变化，因此按版本治理从 V2.4 升级到 V2.5，而不是静默改写历史候选。

V2.4 的版本治理、内容审核、发布门禁、六类轻触点、频控、半屏与安全约束全部沿用。V2.5 只重开“邀请—进入—退出”期间的播放行为：邀请仍轻盈、不打断；用户主动进入后暂停，以避免边看边答造成认知竞争和错过原视频证据。

## 2. 锁定状态矩阵

| 时刻 | V2.4 历史口径 | V2.5 最新口径 |
|---|---|---|
| 轻邀请曝光 | 视频继续播放 | 不变：视频继续播放 |
| 用户忽略邀请 | 视频继续播放；节点进入时间轴 | 不变 |
| 用户点击进入 | 视频继续播放 | 记录进入前状态和当前位置，随后调用一次 `pause()` |
| 用户交互期间 | 视频和声音继续 | 视频保持暂停；不改静音、音量、倍速或进度 |
| 完成、跳过或关闭 | 不控制播放 | 进入前正在播放则从暂停点续播；进入前已暂停则保持暂停 |
| 回看证据 | 只在用户明确操作时跳转 | 不变：只有用户明确点击回看才允许 seek |

## 3. 不变量

- 轻邀请出现、收起和进入时间轴均不得自动暂停。
- 进入财包不自动 seek；暂停位置以点击进入时的媒体时间为准。
- 自动恢复相对暂停位置的漂移不超过 250ms。
- 自动暂停和恢复不得修改 `muted`、`volume` 或 `playbackRate`。
- 无蒙层，半屏最高 48vh，作者头像与财包身份分离。
- 同时最多一个交互；自动邀请最多 4 次、间隔至少 45 秒。
- 完成、跳过和关闭走同一退出策略；重复事件必须幂等。

## 4. 契约与事件影响

播放器与财包容器之间至少需要维护：

- `wasPlayingBeforeInteraction: boolean`：进入前是否正在播放；
- `pausePositionMs: number`：进入时记录的媒体位置；
- `interactionPlaybackState: idle | pausing | paused | restoring`：防止重复打开/关闭产生竞态。

学习事件继续记录 `cue_expand`、`cue_complete`、`cue_skip` 和 `cue_close`。实现阶段新增或扩展 `playback_transition` 事件，记录动作、原因、位置和是否恢复；该事件用于排障与验收，不得被包装成学习分数。

## 5. 新增验收

| ID | 场景 | 预期 |
|---|---|---|
| T-PLAY-01 | 视频正在播放，用户点击进入财包 | 当前位置自动暂停；无 seek；面板 ≤48vh；无蒙层 |
| T-PLAY-02 | 从 T-PLAY-01 完成、跳过或关闭 | 从同一位置恢复播放，漂移 ≤250ms |
| T-PLAY-03 | 视频本来已暂停，用户进入并退出 | 交互期间和退出后均保持暂停 |
| T-PLAY-04 | 连续点击打开/关闭或触发重复事件 | pause/play 至多各执行一次，状态与事件不重复计数 |

四个用例还必须断言音量、静音和倍速不变。轻邀请曝光需有独立用例证明不会暂停。

## 6. 当前实现差距
