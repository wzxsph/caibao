# 财经推演室｜PRD V2.6 → V2.7 差异

状态：Review Candidate 配套记录  
日期：2026-07-23

## 1. 变更原因

V2.6 面向固定四条、本地 `internal_poc` 和 manifest 其余条目 `UNMAPPED`。用户随后明确要求：使用清单全部 25 条、公开部署、展示作者与原作品、前端只保留推荐流和作者页，并取消“自动触点最多 4 个”。这些均改变产品范围、部署边界与验收，因此升级为 V2.7 候选。

## 3. 不变口径

- 入口曝光继续播放；点击财包后暂停；退出按进入前状态恢复。
- 不自动 seek，不改变静音、音量或倍速。
- 半屏最高 48vh、无蒙层、作者头像与财包身份分离。
- 自动邀请至少间隔 45 秒，同时最多一个。
- 六类轻交互、条件语言、无总分、无投资建议。
- 模型只能生成候选；生产 Approved 仍需要证据、权利、财经和安全审核。
- Markdown 是 PRD 唯一源；按用户要求不做 PDF 视觉验收。

## 5. 风险变化

V2.7 将媒体放到公网 Release，风险从“本地目录到期自动空”扩大为“Release 直链仍可能存在”。因此：

1. 2026-08-22 前必须取得续期或执行 Release 下架。
2. 前端到期空态不能代替删除资产。
3. 用户直接要求公开不等于项目完成独立法律核验。
4. 标题 Mock 不能被搜索引擎或评审材料误写成真实多模态财经结论。

## 6. 版本结论

V2.6 转为历史候选；V2.7 成为最新 Review Candidate。V2.0 仍是已批准基线，用户直接裁决即时覆盖相反旧条款。未完成真实联合评审前，不创建 `prd-v2.7-approved` 标签。
