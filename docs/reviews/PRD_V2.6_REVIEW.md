## 5. 分角色检查清单

### 产品与设计

- [ ] POI 足够轻盈，44px/216px/4—6 秒约束清晰且不遮挡作者、字幕、进度条。
- [ ] POI 是页内交互按钮，不会被误解为跳出视频的外链。
- [ ] 半屏 ≤48vh、无蒙层；作者头像与财包身份严格分离。
- [x] 空目录、排除单条和授权到期均有明确、不诱导的空态（`b51e0a50` 自动化证据）。

### 研发与架构

- [x] manifest v1/v2 是唯一媒体源，固定四 ID 交集是唯一推荐集合；当前 25→4、其余 21 条 `UNMAPPED`，没有替补或旧 mock fallback。
- [x] 路径 containment、bytes/SHA-256、FFprobe、授权期限和派生媒体校验可测试。
- [x] Catalog 不暴露本机路径、Cookie、权利附件、远程头像或伪造互动数字。
- [x] GET/HEAD/Range、404/410/416 契约完整。
- [x] pause/release 请求幂等，不复用会强制改变 volume 的通用播放方法。
- [ ] `internal_poc` 在生产环境被硬拒绝，不能通过查询参数绕过。

### 测试

- [x] 四个目标视口覆盖 POI 尺寸、单行省略、触控命中、半屏上限和作者位。

- [x] v1/v2 兼容、未知 schema、新增/重排条目、重复、路径越界、过期、缺文件、bytes/SHA/时长不符全部 fail closed；schema 不扩权。
