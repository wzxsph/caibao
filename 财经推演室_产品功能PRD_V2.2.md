## 二、需求概述

### 1. 需求背景

财经内容的主要问题不是信息不足，而是用户“听懂了句子，却无法运行逻辑”。传统视频、摘要和思维导图通常交付结论，用户容易记住“降息利好股票和黄金”，却忽略融资成本、盈利预期、实际利率、相对利差、避险情绪与提前计价等中间机制。一旦条件变化，用户会把“可能”误用为“必然”。

当前项目已经完成 V0.8 真实视频内容包版可交互 Demo：接入约 4 分 16 秒的本地真实视频测试素材，视频元数据、章节、字幕提示、概念、因果边、6 类帮练节点、证据时间戳与报告配置均从独立 `VideoContentPackage` 读取。页面支持按内容包 ID 选择视频、按内容包版本隔离本地会话、刷新后恢复播放进度与练习状态，并已加入浏览器语音识别入口。当前内容包仍为 `draft`，字幕与知识金标仅完成 OCR 初校；AI、数据库、账户、服务端证据检索和运营审核台尚未接入。

竞品/替代方式的共同缺口如下：视频摘要仍是被动接收；思维导图难以表达成立条件和反向力量；题库偏对错与记忆；通用 AI 问答可能脱离原视频；行情或投顾产品目标不同且存在合规风险。财经推演室的差异是“先预测—再运行条件—用反例校准—主动复述—生成下一步行动”。

### 2. 需求目标

P0 目标：用一条经授权的固定财经视频跑通可评审、可恢复、可度量的理解闭环，并确保模型或网络不可用时仍可完成核心体验。

| 目标类别 | 指标 | P0 建议目标 |
|---|---|---:|
| 北极星 | 有效理解闭环完成率 | ≥ 25% |
| 进入 | 财包伴学开启率 | ≥ 35% |
| 作答 | 强节点平均作答率 | ≥ 65% |
| 干扰 | 连续跳过率 | ≤ 25% |
| 探索 | 人均有效推演次数 | ≥ 2 次 |
| 理解 | 因果复述关键节点命中率 | 比摘要对照组提升 ≥ 20 个百分点 |
| 边界 | “可能→必然”错误率 | 前后测下降 ≥ 30% |
| 安全 | 正确理解“非市场预测”的比例 | ≥ 90% |
| 稳定 | P0 最短路径完成率 | ≥ 99%（演示环境） |

### 3. 受众角色

- 学习用户：观看财经视频、参与推演、查看复盘。
- 内容运营/审核：维护视频、字幕、概念、因果边、题目、证据和发布状态。
- 产品与研究人员：查看漏斗、学习效果与异常数据。
- 系统管理员：管理权限、内容版本、功能开关与审计日志。

### 4. 范围界定

**当前已实现：** 一条本地真实视频测试素材；独立内容包注册表；6 类时间轴帮练；5 个概念；3 条带条件、阻断因素和证据时间戳的因果边；三种国家/政策情景；二选一反馈；文字与浏览器语音复述；个人推演复盘；本地会话恢复；静态规则兜底。

**后续 P0 做：** 完成视频与创作者授权确认；逐句字幕、章节、题目和证据时间戳人工复核；将内容包状态从 `draft` 推进到 `reviewed`；接入事件埋点、服务端会话、审核发布与功能开关；接入受约束的 AI 评价和解释；完善异常与兼容性。

**本次不做：** 任意视频自动发布；实时行情；具体涨跌幅、目标价、买卖或仓位建议；完整宏观仿真；服务端语音转写与语义评分；开放式无限追问；创作者自助后台；跨视频对照；长期知识图谱；社交排名；以单一分数判定整体财经水平。

### 5. 术语定义

| 术语 | 定义 |
|---|---|
| 财包 | 财经推演室的 AI 帮练助手 IP，不是分析师或投资顾问 |
| 强节点 | 到达关键时间点后展示邀请；用户主动进入才暂停视频的练习点 |
| 因果边 | 两个知识节点间的方向关系，必须带条件、反向因素与证据 |
| 情景推演 | 根据审核规则运行变量组合，输出路径和竞争力量，不预测真实市场 |
| 有效推演 | 用户改变至少一个变量并主动点击运行，系统成功生成结果 |
| 有效理解闭环 | 至少完成预测、推演、反例、复述并查看报告 |
| `offered` | 已向用户展示过邀请的节点 |
| `seen` | 用户已完成或主动跳过的节点；当前 Demo 将两者合并，P0 应拆分 |
| 内容包 | 视频、字幕、概念、因果边、题目、规则、证据与版本信息的集合 |
| 状态枚举 | `watching`、`predicting`、`mapping`、`simulating`、`challenging`、`retelling`、`reporting` |

## 三、用户分析

### 1. 目标用户/角色

核心用户为 18—30 岁财经入门型深度内容消费者：常看财经科普、收藏多但复习少，希望低门槛理解术语与传导机制。次级用户为商科学生、职场新人和轻度投资关注者：已有基础但容易机械套结论。非目标用户为寻求实时行情、个股推荐、收益承诺或量化预测的人群。

### 2. 用户场景与用户故事

- 作为财经入门用户，我希望在作者揭晓前先做一次判断，以便发现自己是否真的理解。
- 作为容易套结论的用户，我希望改变经济与情绪条件，以便看到同样降息为何可能出现不同结果。
- 作为学习用户，我希望点开因果节点查看原视频证据，以便知道解释来自哪里。
- 作为学习用户，我希望用三句话复述并获得反馈，以便确认自己能独立讲清逻辑。
- 作为回访用户，我希望刷新或短暂离开后恢复进度，以便继续完成学习闭环。
- 作为内容审核员，我希望每条因果边都有条件、反向因素、时间戳和审核状态，以便控制事实与合规风险。
- 作为产品运营，我希望看到节点曝光、进入、跳过、推演和报告转化，以便优化干扰强度与内容质量。

## 四、整体方案说明

### 1. 业务流程图

```text
进入视频 Feed
  ├─ 视频失败 → 静态内容页 → 可继续进入推演
  └─ 正常播放 → 命中关键时间点 → 展示财包邀请（不自动暂停）
                       ├─ 关闭/跳过 → 继续播放 → 后续邀请降频
                       └─ 主动进入 → 暂停并记录原位置
                                      ↓
                           预测/概念/因果链练习
                                      ↓
                             原位置续播或知识导航
                                      ↓
                   选择预设/调整四变量 → 点击“运行这组条件”
                       ├─ 规则或网络失败 → 静态审核结果兜底
                       └─ 成功 → 路径、冲突、条件、证据、边界
                                      ↓
                         反例初答 → 补充遗漏条件
                                      ↓
                          三句话复述 → 七维反馈
                       ├─ 输入不足 → 明确提示并允许修改
                       └─ 成功 → 生成个人理解报告
                                      ↓
                         定向回看 / 迁移挑战 / 退出
```

### 2. 页面结构/信息架构

```text
财经推演室
├─ 视频 Feed
│  ├─ 视频画面、字幕、章节、进度和社交操作
│  ├─ 财包邀请卡与节点进度
│  └─ 播放控制、时间轴跳转
├─ 财包练习浮层
│  ├─ 名词预检
│  ├─ 预测下一步
│  ├─ 因果链补全
│  ├─ 变量推演
│  ├─ 反例挑战
│  └─ 主动复述
├─ 知识导航（P0）
│  ├─ 5 个概念
│  ├─ 2 条因果链
│  └─ 时间戳证据
├─ 理解报告
│  ├─ 中心变量、三条路径与反向力量
│  ├─ 能力反馈与薄弱点
│  └─ 回看、迁移、复习建议
└─ 内容审核/配置（P0 内部能力，可先配置文件实现）
```

### 3. 全局规则

- 状态：任何页面均覆盖默认、加载中、空数据、异常、无权限、禁用和已选中状态。
- 返回：练习关闭后回到原播放位置；片段回看结束后回到发起任务。
- 邀请：命中节点只展示底部邀请，不自动暂停；同一节点单会话最多强展示一次。
- 降频：连续跳过 2 次后，后续邀请降级为弱提示，不阻断观看。
- 数据新鲜度：变量改变后旧推演结果显示“条件已变化，请重新运行”，不得伪装成新结果。
- 搜索/排序/分页：用户端 P0 无分页与搜索；审核端默认按更新时间倒序，20 条/页，支持标题、内容包 ID、状态筛选。
- 弹窗：高风险动作（发布、撤回、覆盖版本）需二次确认；普通学习操作不使用确认弹窗。
- 空状态：说明原因并提供唯一下一步，不展示空白容器。
- 错误恢复：保留用户输入与变量，不因接口重试清空；同一提交带幂等键。
- 表达：结果统一使用“可能、支撑、承压、冲突、信息不足”，禁止“必涨/必跌”。

## 五、功能详细需求

### 1. 视频 Feed 与财包入口

**页面入口：** Web 首页 `/`；报告演示入口 `/?view=report` 仅供 Demo/测试，正式环境受开关控制。

**页面原型链接：** 当前无外部 Figma/墨刀链接；以运行中的 V0.8 页面、`app/page.tsx` 与 `app/douyin.css` 为原型基线，正式链接待 UI 补充。

**页面元素：** 竖屏真实视频区、抖音精选风格顶栏、章节标签、字幕提示、播放热区、互动栏、全屏观看按钮、时间轴、6 类节点、财包邀请卡、伴学进度、底部导航、免责声明。

**操作逻辑：** 默认自动播放；点击画面切换播放/暂停；到节点时展示邀请且继续播放；点击“叫醒财包/一起看看”暂停并进入对应练习；点击关闭仅收起本次邀请；点击时间轴节点直接进入练习；完成/跳过后从节点后 1 秒续播；播放结束且满足报告门槛时进入报告，否则展示补做或直接查看报告入口。

**数据规则：** 当前内容包 ID 为 `fed-rate-global-capital-001`，版本 `2026-07-22.2`，视频时长 256.167 秒、编码画面 400×224；6 个节点位于 00:08、00:35、01:28、02:32、03:08、03:58。章节、字幕提示、概念、因果边、练习、证据与报告均来自独立本地内容包。P0 改为审核后内容包接口返回，时间精度到秒，正式字幕须带起止时间。

**状态规则：** 首屏骨架屏；视频加载失败显示封面、错误说明和“进入静态推演”；无字幕时隐藏字幕框并记录异常；节点已完成、已跳过、仅曝光使用不同标识；离线时使用已缓存内容包。

**校验规则：** 节点时间须在视频时长内且严格递增；正式长视频相邻强节点建议 ≥120 秒（当前 4 分钟演示内容包例外）；无标题、无授权状态或无审核内容包不得发布。内容包版本变化时，本地旧会话必须失效。

**提示文案：** 视频失败：“视频暂时无法播放，你仍可用文字与图解完成推演。”；恢复成功：“已恢复到上次学习位置。”；关闭邀请不展示打扰性 Toast。

**接口关联：** `GET /api/content-pack/:id`、`POST /api/session`、`PATCH /api/session/:id/progress`、`POST /api/events/batch`。

### 2. 名词预检与预测节点

**页面入口：** 财包邀请卡或时间轴节点。

**页面元素：** 财包头像与对话、场景题、A/B 两个选项、反馈卡、主按钮、“先继续看”。P0 正式预测可扩展为 3 个业务选项、“我不确定”和“先继续看”；Demo 保持双选以降低演示负担。

**操作逻辑：** 选择后立即展示反馈，不自动关闭；允许改选时记录首次答案与最终答案；点击主按钮完成并续播；跳过不判错。反馈顺序固定为“用户判断—合理部分—缺失条件—视频证据—条件变化”。

**数据规则：** 题目、选项、答案解释、证据时间戳与审核版本来自内容包；用户首次作答不可被后续改选覆盖。

**状态规则：** 未选择时主按钮禁用；提交中锁定选项；接口失败使用静态解释；无证据时不展示未经审核的模型解释，并提示“证据暂不可用”。

**校验规则：** 必须选择有效 option_id；重复提交按 `session_id + node_id + attempt_no` 幂等；选项文案建议 ≤40 字。

**提示文案：** “这条路说得通，但还要补一个条件。”；失败：“刚才的回答已保留，解释加载失败，请重试或继续看。”

**接口关联：** `POST /api/nodes/:id/attempts`、`GET /api/nodes/:id/feedback?attempt_id=`。

### 3. 因果链与知识导航

**页面入口：** 因果练习、片尾知识导航、报告薄弱点、证据回看。

**页面元素：** 5 个概念卡、2 条链路、关系箭头、条件标签、打断因素、证据时间戳、来源与审核状态。

**操作逻辑：** 点击节点看通俗定义；点击边看关系、成立条件和反向力量；点击时间戳打开片段模式；拼图完成后先显示路径再显示资产方向。

**数据规则：** P0 概念为政策利率、融资成本、股票估值、黄金机会成本、相对利差。股票链与黄金链的每条边必须绑定至少 1 条证据与 1 个成立条件。

**状态规则：** 边未审核或证据缺失时对用户不可见；图谱为空则回退为审核后的文字卡；片段加载失败保留当前任务并展示文字证据。

**校验规则：** 禁止环路进入 P0 主链；节点 ID 唯一；结果节点不得使用确定性涨跌词；证据时间戳必须落在视频范围内。

**提示文案：** “先别背答案，我们看看中间发生了什么。”；“这条关系正在复核，暂不用于推演。”

**接口关联：** `GET /api/content-pack/:id/graph`、`GET /api/evidence/:id`。

### 4. 四变量推演沙盘

**页面入口：** 知识导航、变量推演节点、报告迁移挑战。

**页面元素：** 利率、通胀、增长、避险情绪控件；A/B/C 预设；“运行这组条件”按钮；股票、黄金、美元影响卡；激活路径、竞争力量、主导候选、条件、证据、边界提示。

**操作逻辑：** 用户调整变量后结果标记过期；点击运行才生成新结果；预设载入后仍需主动运行；结果可展开路径与证据；同一组合重复运行读取缓存但仍记录用户行为。

**数据规则：** 利率=上升/不变/下降；通胀=上升/稳定/下降；增长=改善/温和/衰退；避险情绪=升温/平静。规则引擎先计算路径，AI 仅基于规则结果和证据生成解释。

**状态规则：** 未运行时展示教学空状态；运行中按钮禁用并展示进度；结果冲突显示“方向冲突”；信息不足显示缺失变量；模型失败使用规则模板；规则服务失败使用内置 A/B/C 审核结果。

**校验规则：** 四变量必填且必须属于枚举；请求带内容包版本和幂等键；不得输出具体价格、涨跌幅或操作建议。

**提示文案：** “条件已变化，请重新运行。”；“两股力量正在拔河，当前无法给出单一方向。”；“这是知识情景推演，不构成投资建议或市场预测。”

**接口关联：** `POST /api/simulations`、`GET /api/simulations/:id`。

### 5. 反例挑战

**页面入口：** 首次有效推演完成后；也可从时间轴进入 Demo 节点。

**页面元素：** 反常结果情景、初答选项、反馈、补条件任务、证据卡、完成按钮。

**操作逻辑：** 第一阶段判断反常结果能否推翻机制；第二阶段选择或输入被忽略条件；必须完成两阶段才记为完成。跳过不判错，但报告显示“边界意识未验证”。

**数据规则：** P0 示例为“降息当天股市反而下跌”；合理解释包括盈利下修、风险溢价上升或已提前计价，均须来自审核内容。

**状态规则：** 初答后才展示补条件；第二阶段失败可重试；网络失败保留第一阶段答案；内容缺失时跳过该节点并记录 `content_error`。

**校验规则：** 开放输入 2—120 字；选择题必须提交有效 ID；模型不得把单一答案描述为唯一事实。

**提示文案：** “你找到了更强的反向力。”；“局部机制可能仍成立，只是被另一股力量盖过去了。”

**接口关联：** `POST /api/challenges/:id/attempts`。

### 6. 主动复述与七维反馈

**页面入口：** 反例完成后或时间轴复述节点。

**页面元素：** 三句话任务、文本框、浏览器语音输入按钮、清空按钮、字数、提交按钮、反馈维度、示例折叠区、证据提示。

**操作逻辑：** 用户可打字或调用浏览器 Web Speech Recognition 输入；不支持语音识别时回退为文字输入。当前 Demo 只要有非空内容即可完成，并提示建议字数；P0 应恢复最小长度与敏感内容校验，再基于审核 rubric 评价，展示已命中、待补充和一条改写建议，并允许二次提交对比。

**数据规则：** 当前内容包建议最小 24 字、最大 220 字，并使用“美元外溢、本国条件、条件边界”三组关键词作基础反馈；P0 七维为方向、机制、条件、反向变量、不确定性、证据一致性、表达完整性。报告使用真实作答证据，不把关键词命中直接等同于理解。

**状态规则：** 当前 Demo 空输入禁用，达到建议最小字数后显示“答案已可提交”；录音中展示状态，识别失败或浏览器不支持时给出文字回退提示。P0 评估中须保留输入；模型超时后用规则型反馈并标注“基础反馈”；违规投资请求只做边界提示，不进入投资建议回答。

**校验规则：** 18—300 字，去除纯空白；重复文本允许提交但记录；输入内容按隐私策略保存，支持删除。

**提示文案：** “至少写 18 个字，把中间机制说出来。”；“反馈暂时慢了一点，你的原文已保存。”；“我可以解释机制，但不能给出买卖建议。”

**接口关联：** `POST /api/retells`、`POST /api/retells/:id/evaluate`。

### 7. 个人推演复盘报告

**页面入口：** 完成最小学习证据后自动进入；Feed 的“查看报告”；Demo 参数 `/?view=report`。

**页面元素：** 中心变量、股票/黄金/美元三条主路径、反向力量、能力印章、薄弱点、回看时间戳、迁移挑战、复习建议、免责声明。

**操作逻辑：** 点击回看跳到具体时间并在结束后回报告；迁移挑战创建待办；报告允许稍后继续；不以单一分数作为核心结论。Demo 中的分数和“优秀”是演示值，P0 必须替换为证据化评价。

**数据规则：** 报告只能引用本会话真实完成记录；已跳过节点不得计为掌握；每条能力结论可展开查看证据；报告绑定内容包版本。

**状态规则：** 证据不足展示“尚未验证”而非低分；生成失败展示已完成节点和重试；离线展示本地简版；无权限时仅允许当前匿名会话查看。

**校验规则：** 至少存在 1 次有效作答或推演才生成；能力标签必须有 supporting_event_ids；禁止收益、买卖或排名表述。

**提示文案：** “这不是成绩单，而是你本次如何思考的记录。”；“报告生成失败，已完成内容不会丢失。”

**接口关联：** `POST /api/reports`、`GET /api/reports/:id`、`POST /api/review-tasks`。

### 8. 内容审核与发布

**页面入口：** 内部运营后台；若 P0 暂不建设 UI，则通过受控配置文件和发布脚本完成。

**页面元素：** 内容包列表、版本、授权状态、字幕、概念、因果边、题目、规则、证据、审核状态、发布/撤回。

**操作逻辑：** 编辑生成草稿版本；产品/内容审核后才能发布；发布不可覆盖历史版本；撤回后新会话不可获取，存量报告保留版本引用。

**状态规则：** `draft`、`in_review`、`approved`、`published`、`withdrawn`；授权过期自动阻止新发布并告警。

**校验规则：** 每条主因果边必须有证据、条件、反向因素和审核人；无版权证明、含投资建议或确定性预测时禁止发布。

**提示文案：** “发布后将生成不可变版本，确认发布？”；“缺少证据或审核记录，无法发布。”

**接口关联：** `GET/POST /api/admin/content-packs`、`POST /api/admin/content-packs/:id/review`、`POST /api/admin/content-packs/:id/publish`。

## 六、非功能需求

### 1. 权限需求

| 角色 | 用户内容 | 学习记录 | 内容编辑 | 审核发布 | 配置/日志 |
|---|---|---|---|---|---|
| 匿名用户 | 查看 | 仅当前设备/会话 | 无 | 无 | 无 |
| 登录用户 | 查看 | 查看、继续、删除本人记录 | 无 | 无 | 无 |
| 内容运营 | 预览 | 仅脱敏汇总 | 编辑草稿 | 无 | 查看内容日志 |
| 审核员 | 预览 | 仅脱敏汇总 | 评论/退回 | 审核 | 查看审核日志 |
| 管理员 | 全部 | 按授权排障 | 全部 | 发布/撤回 | 功能开关、审计 |

所有写操作服务端鉴权；用户记录按 user_id/session_id 隔离；身份系统只证明身份，不自动证明工作区权限。

### 2. 埋点需求

| 事件 | 触发时机 | 核心属性 |
|---|---|---|
| `video_view` | 视频可见 ≥1 秒 | content_id、version、source |
| `coach_invite_exposure` | 邀请卡可见 ≥500ms | node_id、time、strength |
| `coach_invite_action` | 进入/关闭/跳过 | action、node_id、skip_streak |
| `node_answer_submit` | 首次/改选提交 | node_type、option_id、attempt_no |
| `simulation_run` | 点击运行且返回 | variables、preset、latency、fallback |
| `evidence_open` | 打开证据/回看片段 | evidence_id、from_page |
| `retell_submit` | 提交复述 | length、evaluation_mode、attempt_no |
| `report_view` | 报告曝光 | evidence_count、completion_flags |
| `review_action` | 回看/迁移/复习 | action、target_id |
| `error_exposure` | 用户看到错误 | error_code、page、recoverable |

埋点不得上传复述全文、身份明文或敏感输入；全文分析走业务接口并按隐私规则存储。

### 3. 兼容要求

- 移动端优先，基准视口 375×812；支持 360px—430px 宽度。
- PC 端以手机画布居中展示，核心功能可操作，不要求桌面信息密度重构。
- 浏览器：iOS Safari 最近 2 个大版本、Android Chrome 最近 2 个大版本、桌面 Chrome/Edge/Safari 最近 2 个大版本。
- 支持触控与键盘；关键按钮有可访问名称；不只依赖颜色表达状态。

### 4. 性能要求

- 4G 良好网络下首屏可交互 P75 ≤2.5 秒；静态骨架 ≤1 秒出现。
- 内容包接口 P95 ≤800ms；规则推演 P95 ≤1 秒；AI 解释 P95 ≤5 秒，超过 3 秒展示进度文案。
- 用户操作反馈 ≤100ms；视频跳转误差 ≤1 秒。
- 埋点批量上报，不阻塞主流程；失败最多退避重试 3 次。

### 5. 安全需求

- 财经安全：禁止买卖、仓位、收益承诺和确定性价格预测；固定免责声明全链路可见。
- 内容安全：模型输出必须受审核证据约束，规则结果与自然语言解释分层。
- 数据安全：传输加密；复述文本最小化存储；日志脱敏；用户可删除学习记录。
- 防重复提交：关键写接口使用幂等键，按钮提交中禁用。
- 输入安全：防 XSS、提示注入与越权读取；后台操作有审计记录。

## 九、前置/依赖与风险

### 1. 依赖需求

- 经授权的视频、准确字幕、时间戳和内容审核人员。
- 规则引擎、证据存储、AI 解释/评价能力及静态兜底模板。
- 会话与用户存储；可选 ChatGPT 登录仅用于跨设备与个人记录。
- UI 原型、财包素材版权与移动端可访问性验收。
- Cloudflare Workers/Sites 构建部署链路与运行日志。

### 2. 约束条件

- 当前是内容包驱动的前端 Demo，数据库 Schema 尚未进入正式业务链路；自动测试覆盖生产构建、关键页面文案和真实内容包/视频资产校验。
- 当前 256.167 秒测试视频与 6 个节点服务于集中演示，不代表真实长视频频率；正式内容需控制打断。
- AI 延迟与不确定性不能阻断核心闭环；规则与内容必须先审核。
- 实时赛事提交规则、截止时间等动态信息不在本需求内推断。

### 3. 潜在风险与预案

| 风险 | 影响 | 预案 |
|---|---|---|
| 邀请过多打断观看 | 跳过、流失 | 只邀请不自动暂停；连续跳过降频；A/B 测试节点数量 |
| AI 幻觉或越界建议 | 合规与信任风险 | 规则先行、证据约束、输出过滤、静态兜底、人工抽检 |
| 用户把推演当预测 | 错误决策 | 全链路条件化语言、免责声明、安全理解题 |
| 报告使用演示分数 | 误导用户 | P0 移除固定分数，能力结论绑定真实事件证据 |
| 本地会话被清除或无法跨设备恢复 | 闭环中断 | 当前按内容包版本保存本地会话；P0 增加服务端会话、断点恢复与幂等提交 |
| 视频/证据不可用 | 核心路径中断 | 静态内容与文字证据兜底 |
| 版权或授权到期 | 无法继续分发 | 授权字段、到期告警、自动下架、版本留痕 |
| 埋点污染或漏报 | 指标失真 | 事件字典、Schema 校验、客户端批量与服务端关键事件双记 |

## 十、上线相关

### 1. 灰度策略

按“内部成员 → 受邀测试用户 10% → 50% → 全量”四阶段灰度。每阶段至少观察 24 小时或达到预设样本量；若错误率、跳过率、越界输出或性能超过阈值则停止扩量。首发仅一个审核内容包。

### 2. 开关控制

所有新功能必须支持服务端开关：`coach_enabled`、`simulation_enabled`、`ai_explanation_enabled`、`retell_enabled`、`report_enabled`。开关关闭时不删除数据，用户回退到普通观看或静态规则体验。Demo 直达参数正式环境默认关闭。

### 3. 回滚方案

- 应用回滚：部署平台切回上一稳定构建。
- 内容回滚：内容包发布采用不可变版本，指针切回上一审核版本。
- AI 回滚：关闭 AI 解释/评价，保留规则与静态反馈。
- 数据回滚：数据库迁移须向后兼容；新字段先可空、后补数据、最后收紧约束。
- 体验回滚：关闭财包入口后保留普通视频播放和免责声明。

上线前必须完成：核心路径与异常路径测试、移动端兼容、权限与越权测试、内容审核、版权确认、埋点校验、性能压测、开关演练和回滚演练。

### 3. 常见 Q&A

**Q：这是投资工具吗？** 不是。产品只帮助理解财经内容中的机制、条件与反例，不提供行情预测或投资建议。

**Q：为什么不自动暂停出题？** 用户应保有观看控制权。节点只发邀请，用户主动进入才暂停。

**Q：为什么不能只用 AI 生成所有内容？** 主因果边、证据和安全边界必须审核；AI 负责受约束解释，不负责创造未经验证的财经事实。

**Q：为什么报告不以分数为中心？** 单一分数会掩盖不同能力。报告更关注用户是否说出了机制、条件、反向力量与不确定性。

**Q：当前 Demo 与 P0 最大差异是什么？** Demo 已使用真实视频测试素材、独立内容包和本地会话恢复，但内容仍是未审核 `draft`，语音仅依赖浏览器能力，反馈仍为静态规则。P0 需要完成授权与人工复核，并补齐服务端持久化、埋点、权限、受约束 AI、异常兜底与可回滚发布。
